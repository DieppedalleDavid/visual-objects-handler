voh.objects — Callback de dessin

Personnaliser entièrement l'apparence d'un objet avec une procédure de dessin. Équivalent de SetObjectDrawingCallback() dans Editors Factory.

Le principe

Par défaut, un objet VOH est un simple rectangle coloré (couleur définie par backgroundColor). Le callback de dessin permet de remplacer ce rendu par défaut par votre propre procédure de dessin: vous décidez exactement ce qui est affiché à l'intérieur de l'objet.

C'est exactement le même concept que SetObjectDrawingCallback() dans Editors Factory: vous écrivez une procédure de dessin, et VOH l'appelle automatiquement chaque fois que l'objet doit être redessiné (redimensionnement, déplacement, etc.).

Comparaison avec Editors Factory (PureBasic):
Dans EF, vous créiez une procédure séparée (Runtime Procedure MyDrawing(...)) puis vous la liiez à l'objet par son nom en texte (SetObjectDrawingCallback(Object, "MyDrawing")).
Dans VOH v2, vous créez une fonction JavaScript et vous la passez directement (pas de nom en texte, pas de Runtime). Le reste fonctionne pareil.

Les 4 paramètres de la procédure de dessin

Votre procédure de dessin reçoit toujours 4 paramètres dans cet ordre:

ParamètreTypeDescription
contextCanvasRenderingContext2DLe « pinceau » — c'est avec lui que vous dessinez. Toutes les commandes de dessin passent par ce contexte (fillRect, fillText, arc, strokeRect, etc.).
widthnombreLargeur actuelle de l'objet en pixels. Cette valeur change si l'objet est redimensionné.
heightnombreHauteur actuelle de l'objet en pixels. Cette valeur change si l'objet est redimensionné.
objectIdnombreL'ID unique de l'objet. Utile si vous utilisez la même procédure pour dessiner plusieurs objets différemment.
Espace de dessin: Votre procédure dessine toujours dans un espace qui va de (0, 0) (coin haut-gauche) à (width, height) (coin bas-droit). Vous n'avez pas à vous soucier de la position de l'objet sur le canevas — VOH gère ça pour vous. C'est comme dans EF: vous dessiniez à partir de (0, 0) dans la zone de l'objet.

Deux façons d'écrire la procédure

JavaScript offre deux façons d'écrire votre procédure de dessin. Les deux font exactement la même chose — c'est une question de style.

Style 1 — Procédure séparée (comme dans EF)

Vous déclarez la fonction en dehors, avec un nom, puis vous la passez à l'objet. C'est la même logique que Runtime Procedure dans EF. 

// 1. Déclarer la procédure de dessin (comme dans EF)
function dessinerBouton(context, width, height, objectId) {
    context.fillStyle = '#4a90d9';
    context.fillRect(0, 0, width, height);
    context.fillStyle = 'white';
    context.font = 'bold 16px Arial';
    context.textAlign = 'center';
    context.textBaseline = 'middle';
    context.fillText('Mon bouton', width / 2, height / 2);
}

// 2. L'attacher à l'objet (par référence, pas par nom en texte)
const bouton = voh.objects.create(canvas, {
    drawingCallback: dessinerBouton   // ← on passe la fonction directement
});
Avantage: La même procédure peut être réutilisée pour plusieurs objets. Utilisez le paramètre objectId pour les différencier si nécessaire.

Style 2 — Procédure en ligne (spécifique à JavaScript)

Vous écrivez la fonction directement dans le paramètre, sans lui donner de nom. C'est plus compact mais la procédure ne peut pas être réutilisée pour un autre objet. 

const bouton = voh.objects.create(canvas, {
    drawingCallback: function(context, width, height, objectId) {
        context.fillStyle = '#4a90d9';
        context.fillRect(0, 0, width, height);
        context.fillStyle = 'white';
        context.font = 'bold 16px Arial';
        context.textAlign = 'center';
        context.textBaseline = 'middle';
        context.fillText('Mon bouton', width / 2, height / 2);
    }
});

Utiliser des données externes dans le callback

Un avantage majeur de VOH v2 par rapport à EF: votre procédure de dessin peut accéder directement aux variables de votre script. Dans EF, vous deviez passer les données via le paramètre iData. Ici, la fonction JavaScript « voit » naturellement les variables qui l'entourent. 

let progression = 0.65;   // 65% — variable externe

function dessinerJauge(context, width, height) {
    // Fond gris (barre vide)
    context.fillStyle = '#e0e0e0';
    context.fillRect(0, 0, width, height);

    // Barre de progression (verte) — utilise la variable externe
    context.fillStyle = '#4CAF50';
    context.fillRect(0, 0, width * progression, height);

    // Texte du pourcentage
    context.fillStyle = '#333';
    context.font = 'bold 14px Arial';
    context.textAlign = 'center';
    context.textBaseline = 'middle';
    context.fillText(Math.round(progression * 100) + '%', width / 2, height / 2);
}

const jauge = voh.objects.create(canvas, {
    drawingCallback: dessinerJauge
});

// Plus tard, quand la progression change:
progression = 0.90;
voh.objects.redraw(jauge);  // ← force le redessin avec la nouvelle valeur
Important: Quand une variable externe change, VOH ne le sait pas automatiquement. Vous devez appeler voh.objects.redraw(objectId) pour forcer le redessin. Le callback est rappelé et il lit la nouvelle valeur de la variable.

Partager une procédure entre plusieurs objets

Comme dans EF, vous pouvez utiliser la même procédure pour dessiner plusieurs objets. Le paramètre objectId permet de savoir quel objet est en train d'être dessiné et d'adapter le contenu. 

function dessinerEtiquette(context, width, height, objectId) {
    context.fillStyle = '#f0f0f0';
    context.fillRect(0, 0, width, height);
    context.fillStyle = '#333';
    context.font = '12px Arial';
    context.fillText('Objet #' + objectId, 10, 20);
}

// La même procédure est utilisée pour 3 objets
const obj1 = voh.objects.create(canvas, { drawingCallback: dessinerEtiquette });
const obj2 = voh.objects.create(canvas, { drawingCallback: dessinerEtiquette });
const obj3 = voh.objects.create(canvas, { drawingCallback: dessinerEtiquette });

Méthodes API

Définir le callback

voh.objects.setDrawingCallback(objectId, callback)
Définit ou remplace la procédure de dessin d'un objet. L'objet est immédiatement redessiné avec le nouveau callback. Passer null pour supprimer le callback et revenir au rendu par défaut (rectangle backgroundColor).
ParamètreTypeDescription
objectIdnombreL'ID de l'objet
callbackfonction | nullProcédure de dessin, ou null pour le rendu par défaut
Retourne: boolean — true si l'objet existe, false sinon

Lire le callback

voh.objects.getDrawingCallback(objectId)
Retourne la procédure de dessin actuellement attachée à l'objet, ou null si l'objet utilise le rendu par défaut.
ParamètreTypeDescription
objectIdnombreL'ID de l'objet
Retourne: Function | null

Forcer un redessin

voh.objects.redraw(objectId)
Force le redessin immédiat de l'objet. Indispensable quand votre callback dépend de données externes (variables, compteurs, état du jeu) qui ont changé. Sans cet appel, l'objet continue d'afficher l'ancien dessin.
ParamètreTypeDescription
objectIdnombreL'ID de l'objet à redessiner
Retourne: boolean — true si l'objet existe, false sinon

SituationRedessin auto ?
L'objet est créé avec un drawingCallbackOui

Une variable externe changeNon — appeler redraw()
Voir aussi: Les callbacks de dessin du fantôme de drag (mouseDragPreviewCallback et mouseDragMultiPreviewCallback) sont documentés dans la page Drag souris. Ces callbacks ont la même signature function(ctx, w, h) mais dessinent le fantôme pendant le déplacement, pas l'objet lui-même.
