voh.canvas — Canvas actif

Le canvas actif est une notion purement API: c'est le canvas désigné comme cible par défaut pour les opérations qui n'en précisent pas un explicitement. Ce n'est pas un "focus" au sens interface — les événements souris (survol, clic, etc.) fonctionnent sur tous les canvas en permanence, qu'ils soient actifs ou non.

Actif ≠ focus souris. Un objet sur un canvas non actif peut très bien réagir au survol de la souris et changer de couleur. L'état "actif" n'a aucun impact sur les événements des objets — c'est au développeur de décider dans son code si l'état actif doit influencer le comportement de son éditeur.

Activer un canvas

voh.canvas.setActive(canvasId) → vrai/faux
Définit le canvas actif. Un seul canvas peut être actif à la fois dans toute l'instance VOH. Le canvas précédemment actif est automatiquement désactivé. Si l'option bringCanvasToFrontOnActivation est activée sur la zone parente, le canvas passe aussi automatiquement au premier plan.
ParamètreTypeDescription
canvasIdnombreL'ID du canvas à activer
Retourne: vrai/faux — true si réussi, false si canvas inexistant.
voh.canvas.setActive(canvas2);

Canvas actif actuel

voh.canvas.getActiveId() → nombre ou null
Retourne l'ID du canvas actuellement actif, ou null si aucun canvas n'est actif (par exemple après avoir supprimé le canvas actif sans en activer un autre).
Retourne: nombre — l'ID du canvas actif, ou null.
console.log(voh.canvas.getActiveId());  // ex: 2

Exemple: réagir au survol sans tenir compte de l'état actif

Les événements souris fonctionnent sur tous les canvas indépendamment. Dans cet exemple, deux canvas coexistent et chacun réagit au survol de ses objets sans qu'un clic préalable soit nécessaire. 
// Les deux canvas écoutent leurs événements en parallèle
voh.on('object:mouseEnter', ({ objectId }) => {
    voh.objects.setBackgroundColor(objectId, '#ffcc00');
});

voh.on('object:mouseLeave', ({ objectId }) => {
    voh.objects.setBackgroundColor(objectId, '#ffffff');
});

// Si on veut conditionner à l'état actif, c'est au développeur de le gérer
voh.on('object:mouseEnter', ({ objectId, canvasId }) => {
    if (canvasId === voh.canvas.getActiveId()) {
        voh.objects.setBackgroundColor(objectId, '#ffcc00');
    }
});
