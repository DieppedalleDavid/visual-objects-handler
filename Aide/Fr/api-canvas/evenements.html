canvas:zOrderChanged

Émis quand l'ordre d'affichage d'un canvas change. Déclenché par voh.canvas.setZOrder(), moveUp(), moveDown(), moveToFront() et moveToBack().
PropriétéTypeDescription
canvasIdnombreL'ID du canvas dont l'ordre a changé
zOrdernombreLe nouvel ordre d'affichage (0 = fond)
Note: Les fonctions moveUp() et moveDown() émettent deux événements: un pour le canvas déplacé et un pour le canvas avec lequel il échange sa position.
Pourquoi deux événements avec moveUp() / moveDown() ?

Ces fonctions fonctionnent par échange de position: le canvas ciblé prend la place de son voisin, et le voisin prend la place du canvas ciblé. Les deux canvas changent de zOrder, donc deux événements sont émis.

Exemple concret: vous avez 3 canvas empilés:
— Canvas A (zOrder 0, au fond)
— Canvas B (zOrder 1, au milieu)
— Canvas C (zOrder 2, devant)

Vous appelez voh.canvas.moveUp(canvasA). Le canvas A échange sa place avec le canvas B:
— Canvas B (zOrder 0) ← il descend d'un cran
— Canvas A (zOrder 1) ← il monte d'un cran
— Canvas C (zOrder 2) ← inchangé

Résultat: deux événements canvas:zOrderChanged émis, un pour A et un pour B.
Mise au premier plan automatique à l'activation

Si l'option voh.zone.setBringCanvasToFrontOnActivation(zoneId, true) est activée, un événement canvas:zOrderChanged sera également émis à chaque fois qu'un canvas est activé, car il est automatiquement placé devant tous les autres.

Par défaut cette option est désactivée: activer un canvas ne modifie pas les z-orders. 
voh.on('canvas:zOrderChanged', (data) => {
    console.log('Canvas', data.canvasId, '→ zOrder', data.zOrder);
});

// Appel : voh.canvas.moveUp(canvasA)
// → Événement 1 : Canvas A → zOrder 1 (il monte)
// → Événement 2 : Canvas B → zOrder 0 (il descend)

PropriétéTypeDescription

ynombrePosition Y du point de départ, relative au canevas (en pixels)

canvas:selectionEnded

Émis quand un tracé de rectangle de sélection se termine.
Déclenché soit quand l'utilisateur relâche le bouton souris, soit à la fin d'une voh.canvas.selection.simulate().
Le rectangle fourni est toujours normalisé: x, y, width et height sont toujours positifs, quel que soit le sens du tracé.
PropriétéTypeDescription
canvasIdnombreL'ID du canevas
xnombrePosition X du coin haut-gauche du rectangle final (en pixels, relatif au canevas)
ynombrePosition Y du coin haut-gauche du rectangle final (en pixels, relatif au canevas)
widthnombreLargeur du rectangle (toujours ≥ 0)
heightnombreHauteur du rectangle (toujours ≥ 0)
Note (souris): Cet événement est émis même si le rectangle est de taille nulle (simple clic sans déplacement). Vérifiez data.width > 0 && data.height > 0 si vous souhaitez ignorer les clics simples.

Note (simulation): voh.canvas.selection.simulate() émet toujours cet événement à la fin du tracé, même en mode instantané (duration: 0). En cas d'annulation via cancelSimulation(), l'événement n'est pas émis — la Promise se résout simplement avec false. 
voh.on('canvas:selectionEnded', (data) => {
    // Ignorer les simples clics
    if (data.width === 0 || data.height === 0) return;

    console.log('Rectangle de sélection:');
    console.log('  Position:', data.x, data.y);
    console.log('  Taille:  ', data.width, '×', data.height);
});

canvas:click

Émis quand l'utilisateur clique sur le fond du canevas (zone vide, aucun objet sous le curseur). Si un objet est sous le curseur, c'est object:click qui est émis à la place — les deux ne sont jamais émis en même temps pour le même clic.

x et y sont les coordonnées en pixels relatives au coin supérieur gauche du canevas (pas de la fenêtre du navigateur). Elles correspondent directement aux coordonnées des objets sur le canevas (même repère que voh.objects.setX()).

button indique quel bouton a été utilisé : 'left' (gauche), 'middle' (molette/milieu) ou 'right' (droit).

originalEvent est l'objet MouseEvent natif du navigateur. Il permet d'accéder aux touches modificatrices (ctrlKey, shiftKey, altKey) et à toutes les propriétés DOM standard.
PropriétéTypeDescription
canvasIdnombreL'ID du canevas sur lequel le clic a eu lieu
xnombrePosition X du clic en pixels, relative au coin supérieur gauche du canevas
ynombrePosition Y du clic en pixels, relative au coin supérieur gauche du canevas
buttontexteBouton utilisé : 'left', 'middle' ou 'right'
originalEventMouseEventL'événement DOM natif du navigateur (accès à ctrlKey, shiftKey, clientX, etc.)
// Créer un objet à l'endroit où on clique sur le fond
voh.on('canvas:click', (data) => {
    console.log('Clic sur fond à', data.x, data.y);

    // Créer un objet là où on a cliqué
    voh.objects.create(data.canvasId, {
        x: data.x - 25,  // centré sur le clic
        y: data.y - 25,
        width: 50, height: 50
    });
});

canvas:mousedown / canvas:mouseup

Émis quand un bouton souris est enfoncé (mousedown) ou relâché (mouseup) sur le fond du canevas.

mousedown est émis au moment exact où le bouton est pressé. mouseup est émis quand le bouton est relâché. click est émis après mouseup si le bouton a été enfoncé et relâché au même endroit.

L'ordre d'émission pour un clic simple est : mousedown → mouseup → click.
PropriétéTypeDescription
canvasIdnombreL'ID du canevas
xnombrePosition X de la souris (pixels, relative au canevas)
ynombrePosition Y de la souris (pixels, relative au canevas)
buttontexteBouton concerné : 'left', 'middle' ou 'right'
originalEventMouseEventL'événement DOM natif du navigateur
// Détecter un clic gauche enfoncé sur le fond
voh.on('canvas:mousedown', (data) => {
    if (data.button === 'left') {
        console.log('Clic gauche enfoncé à', data.x, data.y);
    }
});

// Détecter quand le bouton est relâché
voh.on('canvas:mouseup', (data) => {
    console.log('Bouton relâché à', data.x, data.y);
});

canvas:mousemove

Émis quand la souris bouge au-dessus d'une zone vide du canevas (aucun objet visible sous le curseur).

Important : cet événement n'est émis que quand la souris est sur le fond. Dès que le curseur passe au-dessus d'un objet, c'est object:mousemove qui prend le relais. Quand le curseur quitte l'objet et revient sur le fond, canvas:mousemove reprend.

Fréquence : émis à chaque déplacement de souris détecté par le navigateur (typiquement 60 à 120 fois par seconde selon le navigateur et le système). Éviter d'y mettre du code lourd (pas de création d'objets, pas de calculs coûteux).

x et y sont les coordonnées de la souris en pixels, relatives au coin supérieur gauche du canevas (même repère que les positions des objets).
PropriétéTypeDescription
canvasIdnombreL'ID du canevas au-dessus duquel la souris se déplace
xnombrePosition X actuelle de la souris (pixels, relative au canevas)
ynombrePosition Y actuelle de la souris (pixels, relative au canevas)
buttontexteBouton actuellement enfoncé pendant le déplacement (ou 'left' si aucun)
originalEventMouseEventL'événement DOM natif (accès à movementX, movementY pour le delta de déplacement)
// Afficher les coordonnées de la souris en temps réel
voh.on('canvas:mousemove', (data) => {
    document.getElementById('coords').textContent =
        'X: ' + data.x + ' Y: ' + data.y;
});

// Utiliser le delta de déplacement (mouvement relatif)
voh.on('canvas:mousemove', (data) => {
    const deltaX = data.originalEvent.movementX;
    const deltaY = data.originalEvent.movementY;
    console.log('Delta:', deltaX, deltaY);
});

canvas:wheel

Émis quand la molette de la souris est utilisée au-dessus d'une zone vide du canevas.

deltaY est la valeur la plus courante : négatif = molette vers le haut, positif = molette vers le bas. deltaX est utilisé pour le défilement horizontal (pavé tactile, souris spéciales). deltaZ est très rare (certains périphériques 3D).

Les valeurs delta dépendent du navigateur et du système d'exploitation. Elles ne sont pas en pixels mais en "unités de défilement" (environ 100 par cran de molette classique).
PropriétéTypeDescription
canvasIdnombreL'ID du canevas
xnombrePosition X de la souris au moment de la molette (pixels, relative au canevas)
ynombrePosition Y de la souris au moment de la molette (pixels, relative au canevas)
deltaXnombreDéfilement horizontal (négatif = gauche, positif = droite)
deltaYnombreDéfilement vertical (négatif = vers le haut, positif = vers le bas)
deltaZnombreDéfilement en profondeur (rare, souvent 0)
originalEventWheelEventL'événement DOM natif (accès à deltaMode pour l'unité de mesure)
// Zoom avec la molette sur le fond du canevas
voh.on('canvas:wheel', (data) => {
    const zoneId = voh.canvas.getZoneId(data.canvasId);
    if (data.deltaY < 0) {
        voh.zone.zoomIn(zoneId);   // molette vers le haut → zoom +
    } else {
        voh.zone.zoomOut(zoneId);  // molette vers le bas → zoom -
    }
    data.originalEvent.preventDefault();  // empêche le scroll de la page
});

canvas:contextmenu

Émis quand l'utilisateur fait un clic droit sur le fond du canevas (zone vide). Par défaut, le navigateur affiche son menu contextuel natif. Pour le bloquer et afficher un menu personnalisé, appeler data.originalEvent.preventDefault().

Si un objet est sous le curseur, c'est object:contextmenu qui est émis à la place.
PropriétéTypeDescription
canvasIdnombreL'ID du canevas
xnombrePosition X du clic droit (pixels, relative au canevas)
ynombrePosition Y du clic droit (pixels, relative au canevas)
originalEventMouseEventL'événement DOM natif — appeler .preventDefault() pour bloquer le menu navigateur
// Bloquer le menu natif et afficher les coordonnées
voh.on('canvas:contextmenu', (data) => {
    data.originalEvent.preventDefault();  // bloque le menu natif du navigateur
    console.log('Clic droit sur fond à', data.x, data.y);
});
