Utilisation standalone

Comment utiliser VOH dans votre propre page HTML, sans l'éditeur intégré.

L'éditeur intégré (index.html) et le runner (runner.html) sont des outils de développement fournis pour faciliter les tests. Mais le Core VOH est 100% autonome — vous pouvez l'intégrer dans n'importe quelle page HTML.

Ce qu'il faut

Pour faire tourner VOH, il vous faut au minimum 3 choses :

1. Pixi.js — Moteur de rendu
Le fichier Libraries/Pixi/Pixi.js. C'est le moteur graphique GPU utilisé par VOH pour dessiner sur le canvas. Obligatoire.
2. Les fichiers Core
Les 14 fichiers du dossier Core/, chargés dans l'ordre défini par runner.html. Core-Main.js doit toujours être chargé en dernier.
3. Un conteneur HTML
Un <div> dans votre page qui servira de conteneur pour le rendu. VOH prend automatiquement toute la taille de ce <div>.

Exemple complet minimal

Voici une page HTML complète qui utilise VOH sans l'éditeur :

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Mon application VOH</title>
    <style>
        /* Le conteneur prend toute la page */
        * { margin: 0; padding: 0; }
        #container { width: 100vw; height: 100vh; }
    </style>
</head>
<body>

    <!-- Conteneur pour le rendu VOH -->
    <div id="container"></div>

    <!-- 1. Charger Pixi.js -->
    <script src="Libraries/Pixi/Pixi.js"></script>

    <!-- 2. Charger le Core VOH (dans cet ordre !) -->
    <script src="Core/Core-Diagnostics.js"></script>
    <script src="Core/Core-EventEmitter.js"></script>
    <script src="Core/Core-API.js"></script>
    <script src="Core/Core-Zone.js"></script>
    <script src="Core/Core-CanvasSelection.js"></script>
    <script src="Core/Core-CanvasHistory.js"></script>
    <script src="Core/Core-CanvasPage.js"></script>
    <script src="Core/Core-RenderEngine.js"></script>
    <script src="Core/Core-Canvas.js"></script>
    <script src="Core/Core-Objects.js"></script>
    <script src="Core/Core-ObjectSelection.js"></script>
    <script src="Core/Core-SpatialGrid.js"></script>
    <script src="Core/Core-ObjectInteraction.js"></script>
    <script src="Core/Core-Main.js"></script>

    <!-- 3. Votre script -->
    <script>
    (async () => {

        // Créer et initialiser le moteur
        const voh = new VisualObjectsHandler(
            document.getElementById('container')
        );
        await voh.init();

        // Créer une zone
        const zone = voh.zone.create({
            name: 'Ma zone',
            backgroundColor: 'rgba(30, 30, 50, 1.0)',
            gridIsVisible: true,
            gridColor: 'rgba(255, 255, 255, 1.0)',
            gridOpacity: 0.1
        });

        console.log('VOH prêt, zone:', zone);

    })();
    </script>

</body>
</html>

Points importants

Pourquoi async / await ?
voh.init() est une fonction asynchrone (elle prépare des ressources internes). Pour pouvoir utiliser await, votre code doit être dans une fonction async. C'est pourquoi on enveloppe tout dans (async () => { ... })();.

Dans l'éditeur intégré, le runner fait ça automatiquement pour vous — c'est pour ça que vous pouvez écrire await voh.init(); directement dans un script.
Chemins des fichiers
Les chemins src="Core/Core-API.js" sont relatifs à votre page HTML. Si votre page est dans un sous-dossier, ajustez les chemins en conséquence.

Exemple : si votre page est dans MonProjet/app.html et les fichiers Core dans MonProjet/Core/, les chemins restent les mêmes. Mais si votre page est dans MonProjet/pages/app.html, il faudrait écrire src="../Core/Core-API.js".
Taille du conteneur
VOH prend la taille du <div> conteneur. Si le conteneur fait 500×300 pixels, la zone VOH fera 500×300. Si vous voulez que VOH prenne toute la page, donnez au conteneur width: 100vw; height: 100vh;.
Pas besoin de CodeMirror ni TypeScript
Pour une utilisation standalone, seuls Pixi.js et les fichiers Core sont nécessaires. CodeMirror et TypeScript ne sont utilisés que par l'éditeur intégré.

Exemple avec un fichier script séparé

Vous pouvez aussi mettre votre code dans un fichier .js séparé :

<!-- Dans votre page HTML -->
<script src="Libraries/Pixi/Pixi.js"></script>
<script src="Core/Core-Diagnostics.js"></script>
<script src="Core/Core-EventEmitter.js"></script>
<script src="Core/Core-API.js"></script>
<script src="Core/Core-Zone.js"></script>
<script src="Core/Core-CanvasSelection.js"></script>
<script src="Core/Core-CanvasHistory.js"></script>
<script src="Core/Core-CanvasPage.js"></script>
<script src="Core/Core-RenderEngine.js"></script>
<script src="Core/Core-Canvas.js"></script>
<script src="Core/Core-Objects.js"></script>
<script src="Core/Core-ObjectSelection.js"></script>
<script src="Core/Core-SpatialGrid.js"></script>
<script src="Core/Core-ObjectInteraction.js"></script>
<script src="Core/Core-Main.js"></script>
<script src="mon-script.js"></script>

Et dans mon-script.js :

// mon-script.js
(async () => {
    const voh = new VisualObjectsHandler(
        document.getElementById('container')
    );
    await voh.init();

    // Votre code ici...
})();
Attention : Si vous utilisez un fichier .js séparé, vous devez envelopper votre code dans (async () => { ... })(); pour pouvoir utiliser await. Dans l'éditeur intégré, le runner fait ça automatiquement.
