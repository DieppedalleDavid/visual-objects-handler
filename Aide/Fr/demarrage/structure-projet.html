Rôle de chaque fichier principal

index.html — L'éditeur de code

C'est l'interface de développement. Il contient un éditeur de code (CodeMirror) avec coloration syntaxique, 31 thèmes, personnalisation des couleurs, et un menu Fichier pour ouvrir/sauvegarder des scripts. Quand on clique sur Exécuter (F5), le script est envoyé vers runner.html qui l'exécute.

runner.html — La page d'exécution

C'est la page où le code s'exécute réellement. Elle contient un <div id="container"> qui sert de conteneur pour le rendu VOH, et charge automatiquement Pixi.js et les fichiers du Core. Le code à exécuter est récupéré depuis le localStorage du navigateur (stocké par l'éditeur).

Si une erreur se produit pendant l'exécution, un overlay rouge s'affiche avec le message d'erreur, le fichier concerné et le numéro de ligne.

Important : runner.html n'est pas juste un outil de l'éditeur — c'est la page qui fait tourner le moteur VOH. Si vous créez votre propre application, votre page HTML joue le même rôle que runner.html.

Core/ — Le moteur VOH

Les fichiers JavaScript du Core sont le cœur du moteur. Ils sont chargés par runner.html (ou par votre propre page HTML) dans cet ordre obligatoire : 

 1. Core/Core-Diagnostics.js         ← Système de diagnostic
 2. Core/Core-EventEmitter.js        ← Système d'événements interne (on, off, emit)
 3. Core/Core-API.js                 ← Définit la structure de l'API publique
 4. Core/Core-Zone.js                ← Gestion des zones (ZoneManager)
 5. Core/Core-CanvasSelection.js     ← Rectangle de sélection souris
 6. Core/Core-CanvasHistory.js       ← Historique undo/redo par page
 7. Core/Core-CanvasPage.js          ← Gestion des pages (multi-pages)
 8. Core/Core-RenderEngine.js        ← Moteur de rendu Pixi.js
 9. Core/Core-Canvas.js              ← Gestion des canvas (CanvasManager)
10. Core/Core-Objects.js             ← Gestion des objets (ObjectManager)
11. Core/Core-ObjectSelection.js     ← Sélection d'objets
12. Core/Core-SpatialGrid.js         ← Grille spatiale pour le hit-test
13. Core/Core-ObjectInteraction.js   ← Interaction souris (drag, clic, curseurs)
14. Core/Core-Main.js                ← Classe principale, point d'entrée du moteur
L'ordre de chargement est important. Core-Main.js doit toujours être chargé en dernier car il utilise les classes définies dans les autres fichiers.

Libraries/ — Les dépendances

VOH utilise Pixi.js pour le rendu GPU (obligatoire pour le Core). CodeMirror est utilisé uniquement par l'éditeur intégré — il n'est pas nécessaire si vous créez votre propre interface. TypeScript est optionnel et permet de compiler du TypeScript en JavaScript à la volée.

Medias/ — Les ressources

Dossier pour stocker les images, textures et autres fichiers utilisés par vos scripts. Le sous-dossier Images/ contient les images. Les chemins dans les scripts sont relatifs à runner.html (ou à votre page HTML).

Par exemple, pour utiliser une image dans un script : 

// Le chemin est relatif à runner.html (à la racine du projet)
voh.zone.setBackgroundImage(zone, 'Medias/Images/Zelda.png');

Exemples/ — Les scripts d'exemple

Scripts prêts à l'emploi pour apprendre l'API. Ouvrez-les avec Fichier → Ouvrir dans l'éditeur et appuyez sur F5 pour les exécuter.
