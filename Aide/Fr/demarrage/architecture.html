Architecture

Le moteur VOH est organisé en 7 couches empilées et son API en sous-objets logiques.

Les 7 couches

Zone   (1)  → Conteneur de premier niveau (fond, grille, bordure, viewport)
Canvas (5)  → Surface de travail avec 5 couches superposées :
               couleur de fond, image de fond, grille, objets, overlay
Ui     (1)  → Interface utilisateur (futur éditeur visuel)

Organisation de l'API

L'API est organisée en sous-objets :

voh.zone                 → Gestion des zones
voh.canvas               → Gestion des canvas
voh.canvas.selection     → Apparence du rectangle de sélection
voh.canvas.history       → Historique undo/redo (page active)
voh.canvas.pages         → Gestion des pages (navigation, propriétés, sous-objets)
voh.objects              → Gestion des objets visuels
voh.objects.selection    → Sélection d'objets
voh.on/once/off           → Abonnement aux événements
voh.diagnostics          → Rapport de diagnostic (erreurs, appels API, événements)
voh.connections          → Liaisons entre objets (à venir)
