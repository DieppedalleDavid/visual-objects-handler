Éditeur et exécution

Comment écrire du code et l'exécuter avec l'éditeur intégré.

L'éditeur intégré (index.html)

Le projet inclut un éditeur de code intégré dans index.html. C'est un environnement complet pour écrire et tester vos scripts VOH sans aucun outil externe.

Il offre :

Fonctionnalités de l'éditeur
Édition de code — Coloration syntaxique JavaScript/TypeScript, numéros de ligne, fermeture automatique des parenthèses et crochets, correspondance des accolades.

31 thèmes — 12 thèmes clairs et 19 thèmes sombres, plus un thème entièrement personnalisable avec 14 couleurs ajustables.

Menu Fichier — Nouveau script, Ouvrir un fichier .js depuis le disque, Sauvegarder (Ctrl+S), Sauvegarder sous.

Personnalisation — Couleur de fond de l'éditeur, couleur de l'interface (barres), taille de police, le tout sauvegardé automatiquement.

Exécution — Bouton ▶ Exécuter (F5) pour lancer le script.

Support TypeScript — Sélectionnez « TypeScript » dans le menu déroulant Langage en haut à gauche. Le compilateur TypeScript est intégré : votre code TS est automatiquement compilé en JavaScript avant l'exécution, sans aucune installation supplémentaire.

Comment exécuter un script

Méthode 1 : Écrire et exécuter

Ouvrez index.html dans votre navigateur. Écrivez votre code dans l'éditeur, puis cliquez sur ▶ Exécuter (F5) ou appuyez sur la touche F5.

Méthode 2 : Ouvrir un exemple et exécuter

Cliquez sur Fichier → Ouvrir, naviguez jusqu'au dossier Exemples/Fr/, choisissez un script (par exemple Zone/01-zone-basique.js), puis appuyez sur F5.

Comment ça marche en coulisses

Quand vous cliquez sur Exécuter (F5), voici ce qui se passe :

Étape 1 — Stockage du script
L'éditeur (index.html) prend le code que vous avez écrit et le stocke dans le localStorage du navigateur, sous la clé voh_runner_script. Le langage (JavaScript ou TypeScript) est aussi stocké sous voh_runner_language.
Étape 2 — Ouverture du runner
L'éditeur ouvre runner.html dans un nouvel onglet du navigateur.
Étape 3 — Chargement des bibliothèques
runner.html charge automatiquement, dans l'ordre :
1. Pixi.js — Moteur de rendu GPU
2. Core-Diagnostics.js — Système de diagnostic
3. Core-EventEmitter.js — Système d'événements interne
4. Core-API.js — API publique du moteur
5. Core-Zone.js — Gestion des zones
6. Core-CanvasSelection.js — Rectangle de sélection souris
7. Core-CanvasHistory.js — Historique undo/redo
8. Core-CanvasPage.js — Gestion des pages
9. Core-RenderEngine.js — Moteur de rendu Pixi.js
10. Core-Canvas.js — Gestion des canvas
11. Core-Objects.js — Gestion des objets
12. Core-ObjectSelection.js — Sélection d'objets
13. Core-SpatialGrid.js — Grille spatiale (hit-test)
14. Core-ObjectInteraction.js — Interaction souris
15. Core-Main.js — Classe principale
Étape 4 — Exécution du script
runner.html récupère le script depuis le localStorage, l'enveloppe dans une fonction async (ce qui permet d'utiliser await), et l'exécute. Le script a accès au <div id="container"> qui remplit toute la page pour le rendu VOH.
En cas d'erreur
Si le script provoque une erreur, un overlay rouge s'affiche sur la page avec le message d'erreur, le nom du fichier et le numéro de ligne. Les erreurs non attrapées (exceptions et promesses rejetées) sont aussi capturées automatiquement.
TypeScript : Si le langage est TypeScript, runner.html charge automatiquement le compilateur TypeScript (Libraries/TypeScript/TypeScript.js) et compile le code en JavaScript avant de l'exécuter. Aucune installation n'est nécessaire.
