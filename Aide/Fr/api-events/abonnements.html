Événements — Abonnements

S'abonner, écouter une fois, se désabonner, et inspecter les listeners des événements VOH.

Le système d'événements fonctionne sur le principe pub/sub (publication/souscription). Chaque action dans VOH (créer une zone, modifier une propriété, supprimer un objet…) émet un événement. Vous pouvez vous abonner à ces événements pour réagir automatiquement.

Écoute permanente

voh.on(eventName, callback) → voh
Abonne un callback à un événement. Le callback sera appelé à chaque émission de cet événement, jusqu'à ce qu'on se désabonne avec voh.off().
ParamètreTypeDescription
eventNametexteNom de l'événement (ex: 'zone:created', 'zone:resized')
callbackfonctionFonction appelée à chaque émission. Reçoit un objet data avec les détails
Retourne: voh — L'instance VOH, ce qui permet le chaînage.
// Écouter la création de zones
voh.on('zone:created', (data) => {
    console.log('Zone créée:', data.zoneId, data.name);
});

// Écouter les changements de zoom
voh.on('zone:zoomChanged', (data) => {
    console.log('Zoom:', data.zoom, '(avant:', data.previousZoom + ')');
});

Écoute ponctuelle

voh.once(eventName, callback) → voh
Comme voh.on(), mais le callback n'est appelé qu'une seule fois. Après la première émission, il est automatiquement désabonné — pas besoin d'appeler voh.off().
ParamètreTypeDescription
eventNametexteNom de l'événement
callbackfonctionFonction appelée une seule fois
Retourne: voh — L'instance VOH, ce qui permet le chaînage.
// Attendre la première activation d'une zone
voh.once('zone:activated', (data) => {
    console.log('Première zone activée:', data.zoneId);
    // Ce callback ne sera plus jamais appelé
});

Désabonnement

voh.off(eventName, callback) → voh
Retire un callback spécifique d'un événement. Il faut passer exactement la même référence de fonction que celle utilisée avec voh.on(). C'est pourquoi il faut stocker le callback dans une variable si on prévoit de se désabonner.
ParamètreTypeDescription
eventNametexteNom de l'événement
callbackfonctionLa même référence de fonction passée à on()
Retourne: voh — L'instance VOH, ce qui permet le chaînage.
// 1. Stocker le callback dans une variable
const onGridChanged = (data) => {
    console.log('Grille modifiée:', data.property);
};

// 2. S'abonner
voh.on('zone:gridChanged', onGridChanged);

// 3. Plus tard, se désabonner
voh.off('zone:gridChanged', onGridChanged);

Désabonnement global

voh.removeAllListeners(eventName?) → voh
Retire tous les listeners d'un événement donné. Si aucun nom n'est fourni, retire tous les listeners de tous les événements d'un coup. Utile pour un nettoyage complet sans avoir à se souvenir de chaque callback.
ParamètreTypeDescription
eventNametexte(Optionnel) Nom de l'événement. Si omis, tous les listeners de tous les événements sont retirés
Retourne: voh — L'instance VOH, ce qui permet le chaînage.
// Retirer tous les listeners de 'zone:created'
voh.removeAllListeners('zone:created');

// Retirer TOUS les listeners de TOUS les événements
voh.removeAllListeners();

Inspection des listeners

voh.getListenerCount(eventName?) → number
Retourne le nombre de listeners abonnés à un événement donné. Si aucun nom n'est fourni, retourne le nombre total de listeners sur tous les événements.
ParamètreTypeDescription
eventNametexte(Optionnel) Nom de l'événement. Si omis, compte tous les listeners
Retourne: number — Nombre de listeners.
// Combien de listeners sur 'object:moved' ?
console.log(voh.getListenerCount('object:moved')); // ex: 3

// Combien de listeners au total ?
console.log(voh.getListenerCount()); // ex: 12
voh.getEventNames() → Array<string>
Retourne la liste des noms d'événements qui ont au moins un listener abonné. Utile pour le debug — permet de voir rapidement quels événements sont écoutés.
Retourne: Array<string> — Liste des noms d'événements actifs.
// Quels événements sont écoutés ?
console.log(voh.getEventNames());
// ex: ['zone:created', 'object:moved', 'page:added']

Chaînage

Les méthodes on, once, off et removeAllListeners retournent l'instance VOH, ce qui permet de chaîner plusieurs abonnements en une seule expression. 

voh
    .on('zone:created',  (d) => console.log('Créée:',  d.name))
    .on('zone:deleted',  (d) => console.log('Supprimée:', d.name))
    .on('zone:resized',  (d) => console.log('Taille:', d.width, '×', d.height))
    .on('zone:renamed',  (d) => console.log('Renommée:', d.name));

Format des noms d'événements

Tous les événements suivent le format domaine:action avec deux-points comme séparateur. Le domaine indique la catégorie (zone, canvas, object…) et l'action décrit ce qui s'est passé.

DomaineDescriptionExemples
zoneÉvénements liés aux zoneszone:created, zone:resized
canvasÉvénements liés aux canvas (y compris sélection)canvas:created, canvas:selectionStarted
objectÉvénements liés aux objetsobject:created, object:moved
pageÉvénements liés aux pages (y compris undo/redo)page:added, page:undone
connectionÉvénements de connexions (à venir)connection:created, connection:deleted
Voir aussi: La liste complète des événements zone est disponible dans la page Événements zone du menu.
