voh.diagnostics — Système de diagnostic

Le système de diagnostic capture automatiquement les erreurs JavaScript, les avertissements, les appels API et les événements émis. Il permet de générer un rapport complet pour comprendre d'où vient un problème sans avoir à l'expliquer manuellement.

Ce que le rapport contient

Le rapport de diagnostic inclut automatiquement : la version du Core et l'identifiant de l'instance, les informations système (navigateur, OS, écran, mémoire, support WebGL), l'état du projet au moment du rapport (zones, canevas, objets avec leurs compteurs), les 100 dernières erreurs JavaScript capturées (avec stack trace et numéro de ligne), les 100 derniers avertissements console, les 200 derniers appels API (méthode, arguments, résultat, durée en ms), et les 200 derniers événements émis.

Comment l'utiliser

Ajoutez un appel à voh.diagnostics.showReport() ou voh.diagnostics.downloadReport() dans votre script pour afficher ou télécharger le rapport. Vous pouvez aussi utiliser le raccourci clavier Ctrl+Shift+D dans la fenêtre d'exécution (F5), ou cliquer sur le bouton Rapport de diagnostic dans la barre d'erreur quand une erreur se produit. 

// Afficher le rapport dans une fenêtre (avec boutons Copier et Télécharger)
voh.diagnostics.showReport();

// Télécharger le rapport en fichier .txt
voh.diagnostics.downloadReport();

// Copier le rapport dans le presse-papiers
await voh.diagnostics.copyReportToClipboard();

Méthodes

voh.diagnostics.showReport()
Affiche le rapport de diagnostic dans une fenêtre flottante en lecture seule. La fenêtre contient un bouton Copier pour copier le rapport dans le presse-papiers, un bouton Télécharger pour le sauvegarder en fichier .txt, et un bouton ✕ pour fermer. On peut aussi fermer avec la touche Échap ou en cliquant en dehors de la fenêtre. 
voh.diagnostics.showReport();
voh.diagnostics.downloadReport()
Génère le rapport et propose le téléchargement en fichier .txt. Le nom du fichier inclut la date et l'heure (ex: VOH-Diagnostic-2026-03-06_14h30.txt). 
voh.diagnostics.downloadReport();
voh.diagnostics.copyReportToClipboard() → Promise<vrai/faux>
Copie le rapport de diagnostic dans le presse-papiers du système. Retourne true si la copie a réussi, false si elle a échoué (par exemple si le navigateur refuse l'accès au presse-papiers). 
const copié = await voh.diagnostics.copyReportToClipboard();
console.log(copié ? 'Rapport copié !' : 'Échec de la copie');
voh.diagnostics.generateReport() → objet
Génère le rapport complet sous forme d'objet JavaScript (JSON). Utile si vous voulez traiter les données du rapport dans votre code (par exemple les envoyer à un serveur ou les afficher dans votre propre interface). 
const rapport = voh.diagnostics.generateReport();
console.log('Erreurs:', rapport.statistics.totalErrors);
console.log('Appels API:', rapport.statistics.totalApiCalls);
voh.diagnostics.generateReportAsText() → texte
Génère le rapport formaté en texte lisible (le même contenu que le fichier .txt téléchargé). 
const texte = voh.diagnostics.generateReportAsText();
console.log(texte);
voh.diagnostics.getSummary() → objet
Retourne un résumé rapide sans générer le rapport complet. L'objet retourné contient : errors (nombre d'erreurs), warnings (nombre d'avertissements), apiCalls (nombre d'appels API), events (nombre d'événements émis). 
const résumé = voh.diagnostics.getSummary();
console.log('Erreurs:', résumé.errors);     // ex: 2
console.log('API:', résumé.apiCalls);       // ex: 47
voh.diagnostics.clear()
Vide tous les journaux de diagnostic (erreurs, avertissements, appels API, événements). Les compteurs sont remis à zéro. Utile pour repartir sur un état propre avant de tester un scénario précis. 
voh.diagnostics.clear();
console.log(voh.diagnostics.getSummary()); // { errors: 0, warnings: 0, apiCalls: 0, events: 0 }

Capture automatique

Le système de diagnostic capture automatiquement, sans aucun code à écrire :

Les erreurs JavaScript — toute erreur non capturée (exception, TypeError, ReferenceError, etc.) et les Promises rejetées non gérées. Chaque erreur est enregistrée avec son message, le fichier source, le numéro de ligne, la colonne et la stack trace.

Les avertissements — tout appel à console.warn() est intercepté et enregistré. Le message apparaît toujours dans la console du navigateur comme d'habitude.

Les appels API — chaque appel à une méthode VOH (ex: voh.zone.create(), voh.objects.setX()) est enregistré avec les arguments, la valeur de retour, la durée d'exécution en millisecondes, et l'erreur éventuelle.

Les événements — chaque événement émis par le moteur (ex: zone:created, object:moved) est enregistré avec ses données.

Raccourci clavier

Ctrl+Shift+D dans la fenêtre d'exécution (F5) télécharge immédiatement le rapport de diagnostic sans avoir besoin d'ajouter du code.
