Fonctionnalités de l'éditeur

Guide complet des outils d'assistance au développement intégrés dans l'éditeur de code.

Autocomplétion API

L'éditeur propose une autocomplétion intelligente pour toute l'API VOH. Le menu déroulant se déclenche automatiquement quand vous commencez à taper un chemin API.

Comment ça marche

L'autocomplétion fonctionne sur 3 niveaux, en suivant la hiérarchie de l'API :

Niveau 1 — Racine
Tapez v, vo ou voh → le menu propose voh. En validant, un point est ajouté automatiquement et le niveau suivant s'ouvre.
Niveau 2 — Sous-modules
Après voh. → le menu propose init, zone, canvas, objects. Vous pouvez taper pour filtrer (ex: voh.z → filtre sur zone). En validant un sous-module, un point est ajouté et les méthodes s'affichent.
Niveau 3 — Méthodes
Après voh.objects. → le menu propose toutes les méthodes disponibles (create, delete, setName, etc.). Tapez pour filtrer (ex: voh.objects.set → filtre sur les setters).

Validation d'une proposition

Quand vous validez une méthode (avec Entrée, Tab ou un clic), la signature complète est insérée avec les noms des paramètres : 

// Vous tapez: voh.objects.setN → validez "setName"
// Résultat inséré:
voh.objects.setName(objectId, name)

Templates pour les fonctions create

Les trois fonctions create (zone, canvas, objects) insèrent un template multi-lignes complet avec tous les paramètres acceptés et leurs valeurs par défaut : 

// Vous validez "create" dans voh.zone → résultat:
voh.zone.create({
    name: 'Ma zone',
    width: 1500, height: 1000,
    backgroundColor: 'rgba(40, 40, 50, 1.0)',
    backgroundImage: null,
    backgroundImageOpacity: 1.0,
    borderColor: 'rgba(80, 80, 100, 1.0)',
    borderThickness: 2,
    gridWidth: 25,
    gridHeight: 25,
    gridColor: 'rgba(255, 255, 255, 0.15)',
    gridOpacity: 1.0,
    gridIsVisible: true,
    scrollbarIsVisible: true,
    zoom: 1.0,
    bringCanvasToFrontOnActivation: false
})

L'indentation du template s'adapte automatiquement à la position du curseur dans le code. Supprimez simplement les paramètres que vous ne souhaitez pas modifier (ils gardent leur valeur par défaut).

Suppression et filtrage en direct

L'autocomplétion réagit aussi quand vous supprimez des caractères avec Retour arrière. Par exemple, si vous tapez voh.objects.cr puis supprimez le r, la liste se re-filtre sur toutes les méthodes commençant par c.

Barre d'information API

Une barre d'information fixe est positionnée entre l'éditeur et la barre de statut. Elle s'affiche automatiquement quand le curseur texte est positionné sur un nom de fonction API.

Contenu affiché
Ligne 1 : icône ƒ, signature complète de la fonction (ex: voh.objects.setName(objectId, name)), type de retour à droite (ex: → boolean), et un bouton ? pour ouvrir la documentation.

Ligne 2 : description de la fonction, suivie des paramètres avec leur nom (en orange) et leur description (en gris), séparés par des points médians.

Lien vers la documentation

Le bouton ? à droite de la barre d'info ouvre la page de documentation correspondante dans un nouvel onglet. Le raccourci F1 fait la même chose. La sidebar de l'aide se synchronise automatiquement sur la bonne page.

La barre d'info est basée sur le curseur texte, pas sur la souris. Placez simplement le curseur sur n'importe quel mot d'un chemin API (par exemple sur create dans voh.zone.create) pour voir les informations s'afficher instantanément.

Rechercher et remplacer

Le panneau de recherche s'ouvre avec Ctrl+F (recherche seule) ou Ctrl+H (recherche et remplacement).

Options de recherche
Respecter la casse : distingue les majuscules et minuscules.

Mot entier seulement : ne trouve que les mots complets (ex: chercher name ne trouvera pas setName).

Expression régulière : permet d'utiliser des expressions régulières JavaScript dans le champ de recherche (ex: \d+ pour trouver les nombres).

Le compteur d'occurrences s'affiche en temps réel à côté du champ de recherche (ex: 3 / 12 = 3ème occurrence sur 12 trouvées). Le remplacement peut se faire occurrence par occurrence ou en une seule fois avec Tout remplacer.

Thèmes de coloration syntaxique

L'éditeur propose 31 thèmes de coloration syntaxique, plus un thème entièrement personnalisable. Le choix se fait dans Options → Préférences → Thème.

Thèmes clairs (12)

Neo (par défaut), Eclipse, Elegant, Neat, IDEA, XQ Light, Duotone Light, MDN Like, Paraiso Light, Yeti, Juejin, 3024 Day.

Thèmes sombres (19)

Monokai, Dracula, Material, Material Darker, Material Ocean, Material Palenight, Cobalt, Nord, Solarized, Ayu Dark, Ayu Mirage, Oceanic Next, Gruvbox Dark, Tomorrow Night, Blackboard, MBO, Seti, Zenburn, 3024 Night.

Thème personnalisé

En sélectionnant ✎ Personnalisé..., une fenêtre s'ouvre avec 14 couleurs ajustables : fond, texte, curseur, sélection, numéros de ligne, mots-clés, chaînes, nombres, commentaires, fonctions, variables, opérateurs, propriétés et balises. Le résultat est visible en temps réel dans l'éditeur.

Le thème choisi est sauvegardé automatiquement dans le navigateur et restauré à chaque ouverture de l'éditeur.

Minimap

La minimap est un aperçu réduit du code affiché à droite de l'éditeur. Elle permet de visualiser la structure globale du fichier et de naviguer rapidement en cliquant sur la zone souhaitée.

La largeur de la minimap est ajustable en glissant le bord gauche avec la souris. La position de la zone visible est indiquée par un rectangle semi-transparent qui suit le défilement de l'éditeur.

Taille de police

La taille de police est ajustable de 11px à 16px dans Options → Préférences → Taille de police. La valeur par défaut est 13px. Le choix est sauvegardé automatiquement.

Support des langages

L'éditeur supporte deux langages, sélectionnables dans le menu déroulant en haut à gauche :

JavaScript (par défaut)
Le code est exécuté directement. L'autocomplétion et la barre d'info API sont disponibles.
TypeScript
Le compilateur TypeScript est intégré dans le projet (Libraries/TypeScript/TypeScript.js). Le code TypeScript est automatiquement compilé en JavaScript avant l'exécution, sans aucune installation supplémentaire. L'autocomplétion et la barre d'info API fonctionnent de la même manière.
